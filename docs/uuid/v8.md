# UUID v8 (Custom)

UUID v8 allows vendor-specific UUIDs with custom data.

## Usage

```c
#include <uuid/uuid.h>

uuid_t uuid;
char buf[UUID_STRING_LEN];

// Custom 12 bytes (version/variant bits set automatically)
uint8_t custom[12] = {0xDE, 0xAD, 0xBE, 0xEF, 0xCA, 0xFE, 0xBA, 0xBE,
                       0x01, 0x23, 0x45, 0x67};
uuid_v8(&uuid, custom);
uuid_format(&uuid, buf, sizeof(buf));
// e.g. "deadbeef-cafe-babe-8123-456789abcdef"
//       ^^^^^^^^ ^^^^ ^^^^ ^^^^ ^^^^^^^^^^^^
//       custom data     |ver |variant| custom data
//                       8    10xx

// Fully custom (no automatic version/variant)
uint8_t full[16] = {0xFF, 0xFE, 0xFD, 0xFC, 0xFB, 0xFA, 0xF9, 0xF8,
                     0xF7, 0xF6, 0xF5, 0xF4, 0xF3, 0xF2, 0xF1, 0xF0};
uuid_v8_custom(&uuid, full);
```

## Properties

- Version: 8
- Variant: RFC 4122 (10xx) with `uuid_v8()`, custom with `uuid_v8_custom()`
- Timestamp: None (custom)
- Random bits: Custom
- Deterministic: Depends on implementation

## When to Use

- Vendor-specific identifier formats
- Embedding structured data in UUID
- Legacy system compatibility
- Prototyping new UUID versions
